コンストラクターないで使用している関数　confcheck

以下の流れで処理を行うものとしている
基本的にconfigファイルは
server {
	~~
	~~
	location {
		~~
		~~
	}
	location {
		~~
		~~
	}
}

のようになっている。
{}をディレクティブと呼ぶ場合、大きくconfigurationは

server ディレクティブ
location ディレクティブ

になっている。
つまりserverで始まるようなディレクティブ、もしくはlocationで始まるようなディレクティブ（あえて書くならコメント）しか許可せず、
またserverディレクティブ内にのみlocationディレクティブがあると考えられる。
（location内locationに関しては今のところ考えていない）

ファイルを上から読み込めば以下のようになるはずである。
① serverディレクティブで始まるかどうか
② locationディレクティブで始まるかどうか
①②がそれぞれきちんと閉じるかどうか

上記のような流れになっているが、コンフィグファイルに関して初期設定が存在している。
（例えばindexpageはindex.htmlになっているなど）

これに関して、もし値が格納されていないなら格納し直す必要がある。
